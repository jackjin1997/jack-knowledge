# 决策日志框架

完整记录决策过程，为未来复盘提供基础。

## 为什么记录决策

### 1. 澄清思路
写下来的过程本身就是思考和整理

### 2. 防止遗忘
几个月后，你会忘记当时的想法和感受

### 3. 支持复盘
有完整记录才能有效复盘

### 4. 发现模式
积累多个决策后，可以看到自己的决策模式

### 5. 提升能力
通过记录和复盘，持续提升决策能力

## 记录时机

### 最佳时机：决策做出时

- 思路最清晰
- 感受最真实
- 不会遗漏细节

### 也可以：决策前

- 帮助整理思路
- 看清各个选项
- 澄清价值观

## 使用建议

### 1. 趁热记录

决策做出后立即记录，感受最真实。

### 2. 诚实书写

- 这是给自己看的
- 不需要美化
- 包括犹豫、矛盾、恐惧
- 真实才有价值

### 3. 详细但不冗长

- 抓住关键点
- 不需要写论文
- 核心是：情况、过程、决策、原因

### 4. 标记重点

用**加粗**或〇标记特别重要的洞察。

### 5. 定期回顾

- 每季度回顾决策日志
- 看看自己的成长
- 识别决策模式

## 存储组织

### 文件命名

```
YYYY-MM-DD-decision-name.md

示例：
2026-01-23-job-change.md
2026-02-15-mba-decision.md
```

### 目录组织

```
records/
├── 2026/
│   ├── 01-january/
│   │   ├── 2026-01-23-job-change.md
│   │   └── 2026-01-28-apartment-choice.md
│   └── 02-february/
└── 2025/
```

或按类型：
```
records/
├── career/
├── learning/
├── relationships/
└── finance/
```

选择适合自己的方式。

## 在 Cursor 中使用

### 创建新日志

```
"@jack-knowledge/decision-making/frameworks/decision-journal.md

请帮我创建一个决策日志，关于[决策主题]"
```

### 填写日志

```
"使用决策日志模板，
帮我整理刚才的对话和思考"
```

### 回顾旧日志

```
"@jack-knowledge/decision-making/records/YYYY-MM-DD-xxx.md

请帮我回顾这个决策日志"
```
